# Shared session shortcuts

Use this file as the quick reference for repeatable commands across sessions.

## Common workflows

- **Commit changes**: update the relevant files, run the needed checks, commit with a clear message, then push to the active branch.
- **Update all docs**: sync `README.md`, `PROJECT_PLAN.md`, add-on `README.md`, and `CHANGELOG.md` to the current release/state.
- **Push a version**: bump the app/integration/docs version together, run the existing tests, commit, and push the branch.
- **Start a handoff**: update `SESSION_HANDOVER.md` and `PROJECT_PLAN.md` with the current release and next steps.

## Release workflow

1. Update code.
2. Sync docs and tracker files.
3. Run the existing tests.
4. Commit the release.
5. Push the branch.

## Notes

- Keep release versions numeric.
- Use the central repo at `E:\NextCloud\_Projects\GitHub\GithubConfigSync` for shared handoff state.
- Keep `SESSION_HANDOVER.md` current when switching devices.
